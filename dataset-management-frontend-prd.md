# 数据集管理前端 PRD

## 1. 背景

当前系统已有知识库、问答记录和 AI 自进化相关能力，但缺少统一的数据集资产管理能力。为了支持问答评测、模型效果对比、AI 自进化任务和知识库质量分析，需要在资源库中新增数据集管理模块。

本模块面向前端实现，重点覆盖数据集列表、数据集详情、样本在线编辑和文件导入。问答回流由算法侧自动总结后调用后端接口写入数据集，前端只负责展示回流来源的数据。

## 2. 模块定位

数据集管理放在：

```text
资源库 > 数据集管理
```

数据集是基础资产，后续可被 AI 自进化、问答评测、模型评估、知识库效果分析等模块复用。

AI 自进化模块只消费数据集，不负责数据集主流程管理。AI 自进化中可以提供选择数据集、查看任务使用的数据集、将优质问答沉淀到数据集等入口。

## 3. 产品目标

- 支持创建、查看、删除数据集。
- 支持维护数据集基础信息，如名称、描述。
- 支持数据集样本在线新增、编辑、删除。
- 创建数据集时支持选择创建方式：手动创建、上传文件创建。
- 支持通过 Excel、CSV、JSON 导入样本数据。
- 支持展示算法自动回流的样本数据。
- 每条样本数据必须标记来源：用户上传、手动创建、算法回流。
- 当前阶段数据源等同于上传文件，一个数据源只对应一个文件。
- 支持导入时字段映射、数据预览、必填校验和错误提示。
- 为后续 AI 自进化、问答评测、模型评估提供标准化数据输入。

## 4. 术语说明

### 4.1 数据集

数据集是一组问答评测样本的集合。当前阶段主要用于管理评测 Case，包括问题、问题类型、标准答案、参考文档、参考片段等信息。

### 4.2 样本数据

样本数据是数据集中的一条 Case。每条样本至少包含问题、问题类型和标准答案。

### 4.3 算法回流

算法回流是指算法侧基于知识库问答记录自动总结样本，并调用后端接口写入数据集。

前端不提供“从问答记录导入”的弹窗或手动操作入口，只展示来源为 `flowback` 的样本数据。

示例：

| 算法回流字段 | 写入后的样本字段 |
| --- | --- |
| 用户问题 | `question` |
| 系统回答或人工修正答案 | `ground_truth` |
| 用户批量选择的问题类型 | `question_type` |
| 引用文档 | `reference_doc` |
| 引用文档 ID | `reference_doc_ids` |
| 引用片段 ID | `reference_chunk_ids` |

### 4.4 数据源

当前阶段数据源指用户上传的一个文件。

```text
一个数据源 = 一个文件
```

数据源和文件是一对一关系，不支持一个数据源绑定多个文件，也不支持多个文件合并为一个数据源。用户需要导入多个文件时，需要分别创建多条数据源或分别执行多次导入。

### 4.5 创建方式

创建数据集时需要选择创建方式：

| 创建方式 | 前端文案 | 说明 |
| --- | --- | --- |
| `manual` | 手动创建 | 创建空数据集，进入详情页后用户在线新增样本 |
| `upload` | 上传文件创建 | 创建数据集时引导用户下载模版并上传文件导入样本 |

创建方式用于决定新建数据集时的前端流程，不写入 `Dataset` 主表也可以由前端流程或接口参数承载。

## 5. 信息架构

资源库
└── 数据集管理
    ├── 数据集列表
    └── 数据集详情
        ├── 样本表格

## 6. 数据模型

### 6.1 Dataset 字段

Dataset 主表按后端表设计收敛，不在主表中放数据集类型、标签、关联知识库等扩展字段。

| 字段 | 中文名 | 是否必填 | 前端说明 |
| --- | --- | --- | --- |
| `id` | 数据集 ID | 是 | 系统生成 |
| `name` | 数据集名称 | 是 | 创建时必填 |
| `description` | 数据集描述 | 否 | 用于说明数据集用途 |
| `owner_id` | 创建者 | 是 | 数据集创建人 |
| `group_id` | 所属用户组 | 是 | 数据集所属用户组 |
| `created_at` | 创建时间 | 是 | 列表展示 |
| `updated_at` | 更新时间 | 是 | 列表展示 |

前端列表需要展示的样本数量、来源统计、关联知识库等信息，可以由列表接口聚合返回，不要求写入 `Dataset` 主表。关联知识库如需持久化，建议由后端通过独立关联关系承载。

### 6.2 DatasetItem 字段

DatasetItem 的业务字段以现有数据集模版为准，不使用 `query`、`answer`、`reasoning_chain`、`reference_chunks` 作为主字段命名。

| 字段 | 中文名 | 是否必填 | 前端说明 |
| --- | --- | --- | --- |
| `id` | 样本 ID | 是 | 系统生成 |
| `dataset_id` | 数据集 ID | 是 | 归属数据集 |
| `case_id` | Case ID | 否 | 可为空，系统可自动生成 |
| `question` | 问题 | 是 | 用户问题 |
| `question_type` | 问题类型 | 是 | 如事实问答、总结、推理、多跳、操作问答 |
| `ground_truth` | 标准答案 | 是 | 人工确认后的标准答案 |
| `key_points` | 答案要点 | 否 | 用于评测打分 |
| `reference_context` | 参考上下文 | 否 | 答案依据文本 |
| `reference_doc` | 参考文档 | 否 | 文档名称或文档路径 |
| `reference_doc_ids` | 参考文档 ID | 否 | 支持数组或逗号分隔字符串 |
| `reference_chunk_ids` | 参考片段 ID | 否 | 支持数组或逗号分隔字符串 |
| `generate_reason` | 生成依据 | 否 | 展示为“生成依据”，不要展示为“思维链” |
| `is_deleted` | 是否删除 | 否 | 前端默认不展示已删除数据 |
| `source` | 数据来源 | 是 | `upload`、`flowback`、`manual` |
| `source_session_id` | 来源会话 ID | 否 | 若来自算法回流，记录原会话 ID |
| `created_at` | 创建时间 | 是 | 系统字段 |
| `updated_at` | 更新时间 | 是 | 系统字段 |
| `created_by` | 创建人 | 是 | 系统字段 |

字段映射说明：

| 原表设计字段 | 当前 DatasetItem 字段 |
| --- | --- |
| `query` | `question` |
| `answer` | `ground_truth` |
| `reasoning_chain` | 不作为主字段；如需保存可落到 `generate_reason` 或后续扩展字段 |
| `reference_chunks` | `reference_chunk_ids` |

### 6.3 数据源字段

当前阶段只定义文件型数据源。

| 字段 | 中文名 | 是否必填 | 前端说明 |
| --- | --- | --- | --- |
| `id` | 数据源 ID | 是 | 系统生成 |
| `dataset_id` | 所属数据集 ID | 是 | 数据源归属的数据集 |
| `file_id` | 文件 ID | 是 | 与数据源一对一绑定 |
| `file_name` | 文件名 | 是 | 展示上传文件名 |
| `file_type` | 文件类型 | 是 | Excel、CSV、JSON |
| `file_size` | 文件大小 | 否 | 展示文件大小 |
| `import_status` | 导入状态 | 是 | 解析中、导入成功、导入失败 |
| `success_count` | 成功条数 | 否 | 导入成功样本数 |
| `failed_count` | 失败条数 | 否 | 导入失败样本数 |
| `created_by` | 上传人 | 否 | 系统字段 |
| `created_at` | 上传时间 | 否 | 系统字段 |

## 7. 数据来源

### 7.1 用户上传

用户通过 Excel、CSV、JSON 文件导入样本数据。

当前阶段一次导入只选择一个文件，并生成一个文件型数据源。数据源和文件是一对一关系。

导入成功后：

```text
source = upload
```

需要记录数据源 ID、文件 ID 和来源文件名。

### 7.2 手动创建

用户在数据集详情页中在线新增单条样本。

新增成功后：

```text
source = manual
```

### 7.3 算法回流

算法侧基于知识库问答记录自动总结样本，并调用后端接口写入当前数据集。

导入成功后：

```text
source = flowback
```

需要记录原会话 ID 到 `source_session_id`，并保留引用文档和引用片段信息。前端不提供手动回流入口。

## 8. 导入格式

### 8.1 支持格式

MVP 支持：

- `.xlsx`
- `.xls`
- `.csv`
- `.json`

不直接支持 `.numbers` 导入。前端需要提示用户先从 Numbers 导出为 Excel 或 CSV。

### 8.2 模版下载

用户在创建数据集时选择“上传文件创建”后，需要展示三种模版下载入口：

- Excel 模版：`.xlsx`
- CSV 模版：`.csv`
- JSON 模版：`.json`

模版字段需要与导入字段保持一致：

```text
case_id, question, question_type, ground_truth, key_points, reference_context, reference_doc, reference_doc_ids, reference_chunk_ids, generate_reason, is_deleted
```

其中 `question`、`question_type`、`ground_truth` 是必填字段，模版中需要有明确标识或示例说明。

### 8.3 Excel 和 CSV 表头

推荐表头：

```text
case_id, question, question_type, ground_truth, key_points, reference_context, reference_doc, reference_doc_ids, reference_chunk_ids, generate_reason, is_deleted
```

### 8.4 JSON 格式

推荐 JSON 为数组结构：

```json
[
  {
    "case_id": "case_001",
    "question": "如何配置模型供应商？",
    "question_type": "操作问答",
    "ground_truth": "进入模型供应商页面，新增 API Key 并选择默认模型。",
    "key_points": "进入模型供应商页面；新增 API Key；选择默认模型",
    "reference_context": "模型供应商配置说明...",
    "reference_doc": "模型配置手册",
    "reference_doc_ids": ["doc_001"],
    "reference_chunk_ids": ["chunk_001", "chunk_002"],
    "generate_reason": "答案依据来自模型配置手册相关片段",
    "is_deleted": false
  }
]
```

可兼容对象包装结构：

```json
{
  "items": [
    {
      "question": "如何配置模型供应商？",
      "question_type": "操作问答",
      "ground_truth": "进入模型供应商页面，新增 API Key 并选择默认模型。"
    }
  ]
}
```

## 9. 页面需求

### 9.1 数据集列表页

入口：

```text
资源库 > 数据集管理
```

展示字段：

- 数据集名称
- 数据集描述
- 关联知识库
- 样本数量
- 来源统计
- 创建人
- 更新时间

支持能力：

- 关键词搜索。
- 新建数据集。
- 编辑数据集基础信息。
- 删除数据集。
- 进入数据集详情。
- 空状态、加载状态、错误状态。

### 9.2 新建数据集弹窗

表单字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 数据集名称 | 是 | 不能为空 |
| 数据集描述 | 否 | 说明用途 |
| 关联知识库 | 否 | 支持多选，不选择也可以创建 |
| 创建方式 | 是 | 手动创建 / 上传文件创建，放在关联知识库下方 |

创建方式放在“关联知识库”下面：

- 选择“手动创建”时，不展示文件上传区域。
- 选择“上传文件创建”时，在创建方式下方展开 Excel、CSV、JSON 三种模版下载入口和文件上传区域。

选择“手动创建”并提交时：

- 提交后创建空数据集。
- 创建成功后建议直接进入数据集详情页，页面主体展示空样本表格。

选择“上传文件创建”并提交时：

- 一次只允许上传一个文件。
- 用户可以先下载模版，按模版填写后上传。
- 上传并确认导入后创建数据集和文件型数据源。
- 创建成功后进入数据集详情页，页面主体展示导入后的样本表格。

### 9.3 编辑数据集基础信息

编辑对象是数据集元信息，不是样本数据。

支持编辑：

- 数据集名称。
- 数据集描述。
- 关联知识库，支持多选，非必填。

创建方式只记录初始创建流程，不建议在编辑数据集基础信息时修改。

不在此入口编辑样本内容。

### 9.4 数据集详情页

用户从数据集列表点击进入数据集后，页面主体默认直接展示样本表格，不需要再点击“数据明细”Tab。

详情页不重复展示数据集概览信息。数据集名称、描述、关联知识库、样本总量、来源统计、更新时间等信息在数据集列表页展示即可。

详情页顶部只保留返回路径或面包屑，以及样本相关操作。

主要操作：

- 导入数据。
- 手动新增样本。
- 批量删除样本。

辅助入口：

- 导入记录。
- 基础配置。

辅助入口可以使用 Tab、按钮或二级菜单承载，但不能影响用户进入详情页后直接查看和编辑样本表格。

### 9.5 样本表格

展示字段：

- `case_id`
- `question`
- `question_type`
- `ground_truth`
- `reference_doc`
- `source`
- `updated_at`

支持能力：

- 关键词搜索。
- 按问题类型筛选。
- 按来源筛选。
- 新增样本。
- 在线编辑样本。
- 删除单条样本。
- 批量删除样本。
- 点击样本行后展开编辑区。
- 在展开编辑区内统一编辑完整字段。

表格展示规则：

- 长文本默认省略。
- 鼠标悬浮或点击后可查看完整内容。
- 所有字段统一通过行内展开区域展示和编辑。
- 表格列宽固定，避免长文本撑破页面。

### 9.6 样本在线编辑

在线编辑是本模块的核心能力。用户进入数据集详情页后，可以直接对数据集内的样本数据进行在线新增、修改和删除。

在线编辑字段包括：

- `case_id`
- `question`
- `question_type`
- `ground_truth`
- `key_points`
- `reference_context`
- `reference_doc`
- `reference_doc_ids`
- `reference_chunk_ids`
- `generate_reason`
- `is_deleted`

必填规则：

- `question` 必填。
- `question_type` 必填。
- `ground_truth` 必填。

交互建议：

- 表格本身保持只读展示，避免单元格编辑和展开编辑两种样式混用。
- 用户点击样本行后，当前行下方展开统一编辑区。
- 所有可编辑字段都在展开编辑区中编辑，包括短字段和长字段。
- 行编辑态提供“保存”“取消”操作。
- 同一时间建议只允许一行处于编辑态，避免多行未保存内容互相干扰。
- 编辑来源字段只读展示，不允许用户手动修改。
- 保存失败时保留用户已输入内容，并展示错误原因。
- 用户切换编辑行、翻页、筛选、刷新或离开页面时，如有未保存内容，需要二次确认。

### 9.7 手动新增样本

用户在数据集详情页点击“新增样本”。

表单字段同样本字段。

提交规则：

- `question` 不能为空。
- `question_type` 不能为空。
- `ground_truth` 不能为空。
- 新增成功后 `source = manual`。
- 新增成功后刷新列表或插入当前表格第一行。

### 9.8 文件导入弹窗

导入流程：

1. 选择文件。
2. 解析文件内容。
3. 字段映射。
4. 数据预览。
5. 校验错误展示。
6. 确认导入。
7. 展示导入结果。

文件选择阶段：

- 支持拖拽上传。
- 一次只允许选择一个文件。
- 支持下载 Excel、CSV、JSON 三种导入模版。
- 展示文件名、文件大小、文件格式。
- 文件格式不支持时提示错误。
- `.numbers` 文件提示用户先导出为 Excel 或 CSV。

字段映射阶段：

- 自动匹配同名字段。
- 支持用户手动调整字段映射。
- `question` 必须映射。
- `question_type` 必须映射。
- `ground_truth` 必须映射。
- 必填字段未映射时不能进入下一步。

数据预览阶段：

- 展示前 20 到 50 条数据。
- 错误行高亮。
- 展示错误原因。
- 支持只看错误数据。
- 支持返回上一步修改映射。

导入结果阶段：

- 展示成功条数。
- 展示失败条数。
- 展示失败原因。
- 导入成功后刷新样本表格。

### 9.9 导入记录

展示文件导入历史。每条导入记录对应一个数据源，每个数据源对应一个文件。

- 文件名。
- 文件类型。
- 文件大小。
- 导入人。
- 导入时间。
- 成功条数。
- 失败条数。
- 导入状态。
- 失败原因入口。

## 10. 前端校验规则

### 10.1 必填校验

以下字段在新增、编辑、导入时均为必填：

- `question`
- `question_type`
- `ground_truth`

### 10.2 文件导入校验

- 文件类型必须是 Excel、CSV、JSON。
- 表头为空时提示错误。
- 必填字段没有映射时禁止导入。
- 必填字段值为空的行标记为错误。
- JSON 结构不合法时提示解析失败。
- `reference_doc_ids` 支持数组或逗号分隔字符串。
- `reference_chunk_ids` 支持数组或逗号分隔字符串。
- `is_deleted` 支持 `true`、`false`、`1`、`0`、`是`、`否`。

### 10.3 在线编辑校验

- `question` 为空时禁止保存。
- `question_type` 为空时禁止保存。
- `ground_truth` 为空时禁止保存。
- 保存失败时保留用户编辑内容。
- 离开编辑态前，如存在未保存变更，需要二次确认。

### 10.4 展示安全

- 所有用户输入内容按纯文本展示。
- 不渲染用户上传内容中的 HTML。
- 字段编辑统一在行内展开区域完成。
- 表格列宽固定，避免长文本撑破页面。

## 15. 前端验收标准

- 用户可以在资源库中进入数据集管理。
- 用户可以创建一个数据集。
- 用户创建数据集时可以选择“手动创建”或“上传文件创建”。
- 用户选择“上传文件创建”时，可以看到 Excel、CSV、JSON 三种模版下载入口。
- 用户选择“手动创建”时，创建后进入空样本表格。
- 用户可以编辑数据集基础信息。
- 用户可以删除数据集。
- 用户可以进入数据集详情页。
- 用户可以在详情页查看样本列表。
- 用户可以在线新增样本。
- 用户可以在线编辑样本。
- 用户可以删除样本。
- `question`、`question_type`、`ground_truth` 任一为空时无法保存。
- 用户可以上传 Excel、CSV、JSON 文件。
- 用户一次只能上传一个文件。
- 每次文件导入会生成一个数据源记录，且数据源与文件一对一。
- 用户上传 `.numbers` 文件时，前端提示需要先导出为 Excel 或 CSV。
- 导入时可以自动识别字段，并允许手动映射。
- 必填字段未映射时不能继续导入。
- 必填字段值为空的行有明确错误提示。
- 导入后数据来源显示为“用户上传”。
- 手动新增的数据来源显示为“手动创建”。
- 算法回流的数据来源显示为“回流”。
- 表格长文本不会撑破页面。
- 删除和批量删除都有二次确认。
- 无数据、加载中、接口失败都有对应状态展示。

## 16. 需要产品或后端确认的问题

- `question_type` 的枚举值是否固定，还是允许用户自定义。
- `case_id` 是由前端导入保留，还是统一由后端生成。
- 文件导入是前端解析后提交 JSON，还是后端接收单个文件统一解析。
- `reference_doc_ids` 和 `reference_chunk_ids` 后端最终存数组还是字符串。
- 是否需要支持样本去重，去重规则是否按 `question + ground_truth`。
- 数据集删除是否为逻辑删除。
- 样本 `is_deleted` 是否需要暴露给普通用户编辑。
- 算法回流写入数据集时，`question_type` 和 `ground_truth` 的生成规则由算法侧还是后端统一处理。
